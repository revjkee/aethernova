# _common/fixtures

## Назначение

Директория `_common/fixtures` предназначена для хранения **общих тестовых данных**, которые переиспользуются в нескольких наборах тестов
(valid, edge_cases, security, integration).

Эти файлы **не являются самостоятельными тест-кейсами** и не должны запускаться напрямую.
Они служат источником базовых, стабильных и согласованных данных.

## Зачем это нужно

Использование общего слоя фикстур решает следующие проблемы:

- устранение дублирования тестовых данных
- единая точка истины для базовых payload
- предотвращение рассинхронизации между тестами
- упрощение сопровождения и масштабирования test suite
- предсказуемость CI и контрактных тестов

Это стандартный паттерн data-driven тестирования.

## Что допустимо хранить здесь

Разрешено хранить **только универсальные данные**, не привязанные к конкретному сценарию.

Примеры допустимых файлов:
- `valid_login.json` — базовый валидный логин
- `valid_user.json` — стандартный пользователь
- `default_headers.json` — общие HTTP-заголовки
- `base_meta.json` — общие метаданные
- `common_passwords.json` — стандартные тестовые пароли
- `schema_samples.json` — общие примеры схем

## Что запрещено хранить здесь

Не допускается размещать:
- edge cases
- негативные сценарии
- security payloads
- данные с заведомо ошибочными значениями
- данные, предназначенные для одного теста

Такие файлы должны находиться в соответствующих директориях:
`edge_cases/`, `security/`, `invalid/`.

## Правила именования

- имена файлов в `snake_case`
- без указания сценария или типа теста в названии
- один файл — один логический набор данных
