# experiments

## Статус каталога

Данный каталог является архивом экспериментальных скриптов.
Каталог находится в режиме только для чтения с точки зрения логики проекта.
Скрипты, находящиеся здесь, не используются в продакшене, CI или активных тестовых контурах.

Назначение каталога — долговременное хранение исторических экспериментов для анализа эволюции идей, гипотез и технических решений.

## Назначение

Каталог `experiments` используется исключительно для следующих целей:
- фиксация экспериментальных подходов, которые не были доведены до промышленного уровня
- анализ причин отказа от конкретных архитектурных или алгоритмических решений
- восстановление контекста технических решений при аудитах и расследованиях
- обучение и внутренний разбор ошибок проектирования

Любое иное использование считается нарушением регламента проекта.

## Гарантии и ограничения

Гарантируется:
- неизменяемость файлов без явной архивной причины
- сохранение исходного контекста эксперимента
- отсутствие автоматического исполнения

Не гарантируется:
- работоспособность кода
- безопасность исполнения
- совместимость с текущими версиями зависимостей
- соответствие актуальным стандартам качества

## Правила добавления файлов

Добавление новых файлов в данный каталог допускается только при выполнении всех условий:
- файл не используется и не будет использоваться в активной кодовой базе
- файл имеет экспериментальный характер
- файл не содержит секретов, ключей, токенов или персональных данных
- файл сопровождается комментариями, поясняющими цель эксперимента

Каждый эксперимент должен быть самодостаточным и не зависеть от внешнего состояния системы.

## Требования к воспроизводимости

Если эксперимент потенциально воспроизводим, он должен содержать:
- зафиксированные версии интерпретатора и библиотек
- описание входных данных
- ожидаемый результат или критерий остановки
- причину прекращения или отказа от эксперимента

При отсутствии этих данных эксперимент считается невоспроизводимым по умолчанию.

## Требования к безопасности

Запрещено:
- выполнять скрипты из данного каталога в production или staging окружениях
- запускать код без предварительного ручного аудита
- использовать данные эксперименты как основу для решений без повторной валидации

Каталог рассматривается как потенциально небезопасный с точки зрения исполнения.

## Связь с основной кодовой базой

Код из данного каталога:
- не импортируется в активные модули
- не включается в тестовые наборы
- не используется в документации как актуальный пример

Любая попытка прямой интеграции считается архитектурной ошибкой.

## Ответственность

Ответственность за:
- интерпретацию результатов экспериментов
- повторное использование идей
- перенос решений в активный код

лежит на инженере, выполняющем анализ, а не на авторе эксперимента.

## Аудит и сопровождение

Каталог может быть использован при:
- внутреннем техническом аудите
- анализе инцидентов
- ретроспективах архитектурных решений

Удаление файлов допускается только по результатам формального аудита и должно быть задокументировано.

## Итог

Каталог `experiments` является историческим архивом.
Он существует для памяти, анализа и обучения, но не для исполнения.
Любые изменения должны рассматриваться как исключение, а не норма.
