## 📋 Описание

Identity Access Core - критическая система управления идентичностью, аутентификации и авторизации для экосистемы AetherNova. Это базовая система безопасности, от которой зависят все остальные компоненты платформы.

## 🎯 Критические возможности

- **User Authentication** - Аутентификация пользователей с поддержкой bcrypt
- **Access Control** - Контроль доступа на основе ролей (RBAC)
- **Permission Management** - Гибкое управление разрешениями
- **Identity Federation** - Поддержка федеративной идентичности
- **Session Management** - Управление пользовательскими сессиями
- **Token-based Auth** - JWT токены для API доступа

## 🔐 Компоненты системы

### 1. Authentication Service

Обеспечивает аутентификацию пользователей:

```python
from src.authentication import AuthenticationService

auth = AuthenticationService(config)

# Аутентификация
user = await auth.authenticate("username", "password")

# Генерация токена
token = await auth.generate_token(user, expires_in=3600)

# Проверка токена
payload = await auth.verify_token(token)
```

**Возможности:**
- Хеширование паролей с bcrypt
- JWT токены для API
- Блокировка аккаунтов после неудачных попыток
- Управление пользователями
- Экстренный администратор

### 2. Authorization Engine

Контроль доступа на основе ролей и политик:

```python
from src.authorization import AuthorizationEngine, Permission

authz = AuthorizationEngine(config)

# Проверка разрешения
has_access = await authz.check_permission(user, "resource", Permission.WRITE)

# Предоставление разрешения
await authz.grant_permission("user_id", "resource", Permission.READ)

# Создание роли
await authz.create_role("moderator", [Permission.READ, Permission.WRITE])
```

**Возможности:**
- RBAC (Role-Based Access Control)
- ACL для ресурсов
- Гибкие политики доступа
- Предустановленные роли (superuser, admin, user, readonly, guest)

### 3. Session Manager

Управление пользовательскими сессиями:

```python
from src.session_manager import SessionManager

sessions = SessionManager(config)

# Создание сессии
session = await sessions.create_session(user, ip_address="192.168.1.1")

# Проверка сессии
is_valid = await sessions.validate_session(session_id)

# Уничтожение сессии
await sessions.destroy_session(session_id)

# Очистка истекших сессий
await sessions.cleanup_expired_sessions()
```

**Возможности:**
- Автоматическое истечение сессий
- Лимит сессий на пользователя
- Отслеживание IP и User-Agent
- Метаданные сессий

## 🔒 Безопасность

### Экстренный администратор

При инициализации системы автоматически создается экстренный администратор:

- **Username**: `emergency_admin`
- **Password**: Задается в конфигурации (`CHANGE_IMMEDIATELY` по умолчанию)
- **Роли**: `admin`, `emergency`, `superuser`
- **Разрешения**: `*` (все)

⚠️ **ВАЖНО**: Смените пароль экстренного администратора после первого запуска!

### Защита от брутфорса

- Блокировка аккаунта после 5 неудачных попыток входа
- Длительность блокировки: 15 минут
- Автоматическая разблокировка

### Шифрование

- Пароли хешируются с помощью bcrypt (cost factor: 12)
- JWT токены подписываются с помощью HS256
- Ключ шифрования генерируется автоматически

## 🐛 Troubleshooting

### Проблема: Не удается аутентифицироваться

```python
# Проверьте пароль экстренного админа
from config import config
print(config.emergency_admin_password)

# Проверьте наличие пользователя
user = auth_service.get_user("emergency_admin")
print(user)
```

### Проблема: Сессия истекает слишком быстро

```python
# Увеличьте таймаут в конфигурации
IDENTITY_ACCESS_CORE_EMERGENCY_SESSION_TIMEOUT=7200  # 2 часа
```

### Проблема: Заблокирован аккаунт

```python
# Сбросьте счетчик неудачных попыток
auth_service.failed_attempts.clear()
auth_service.locked_accounts.clear()
```
